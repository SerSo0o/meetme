# Code style rule

- Use TypeScript strictly and prefer explicit types at boundaries.
- Prefer small reusable functions and components.
- Avoid dead code and placeholder abstractions.
- Prefer boring, readable naming.
- Keep folder structure scalable.
- Comment only where context is not obvious from the code.
- Keep implementation practical, not clever.
